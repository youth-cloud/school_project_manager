# 项目进程

## 当前阶段
- 项目已进入 `MVP 可交付` 阶段。
- 主业务流程、权限边界、数据一致性、上传下载链路、前端主要页面和部署基础已经完成。
- 当前版本适合用于课程答辩、作品集展示、GitHub 发布和后续继续迭代。

## 项目定位
- 项目名称：学校实训管理平台
- 项目目标：做一个贴近真实校园场景、可演示、可部署、可继续扩展的前后端分离系统
- 业务口径：当前 `批次` 定义为“单教师 + 单课程 + 单班级 + 单次实训”

## 当前已完成

### 1. 基础架构
- 前端完成 `Vue 3 + Vite + TypeScript + Element Plus + Pinia + Vue Router` 基础架构
- 后端完成 `Spring Boot 4 + Spring Security + JWT + MyBatis-Plus + Redis + MySQL` 基础架构
- 已完成登录、获取当前用户、修改密码、管理员重置密码、刷新 token、退出登录
- 已完成管理员、教师、学生三类角色的菜单、路由和接口权限控制

### 2. 业务模块
- 已完成公告、课程、班级、实训批次、课题、选题申请、建组申请、正式项目组、项目组成员
- 已完成阶段任务、阶段提交、提交附件、评审记录、周报、答辩安排、答辩记录、成绩记录
- 已完成用户管理、个人信息维护、账号安全相关页面

### 3. 核心业务规则
- 已完成建组申请审核通过后自动生成正式项目组
- 已完成申请人默认成为组长，其他成员可通过项目组 ID 加入
- 已完成同一课题允许多个项目组选择，不再错误限制为只能一个项目组
- 已完成加入项目组后自动补建并通过选题申请
- 已完成退出项目组后自动取消系统锁题
- 已完成 `selectedCount` 统计修复与历史数据自愈

### 4. 安全与一致性
- 已完成查询范围收口，避免不同角色看到不该看到的数据
- 已完成教师越权写入和撞 ID 修改的边界修复
- 已完成多条主业务链路的删除依赖保护，避免出现孤儿数据
- 已完成审批流、退组、撤回、拒绝等状态一致性收口
- 已完成主要对象的主关系固定化，避免中途改挂父级对象

### 5. 前端交付状态
- 已完成主要业务页的蓝白风格统一
- 已完成全站大部分页面的名称化展示，减少直接显示 ID
- 已完成主要页面乱码清理
- 已完成列表页统一分页接入，`TopicApplication` 页还额外增强了分页可见性
- 已完成多处详情弹窗、按钮状态和交互细节优化

### 6. 部署与工程化
- 根目录已具备 `docker-compose.yml`
- 前后端已具备各自 `Dockerfile`
- 前端已具备 `nginx.conf`
- 后端已具备开发/生产环境配置
- 已补齐根目录 `.env.example`
- 已补齐 GitHub Actions 持续集成配置，可自动检查前后端能否完成构建

## 当前验证结果
- 前端最近一次 `npm run build` 已通过
- 后端此前多轮 `./mvnw -DskipTests package` 已通过
- 本轮已完成主要页面乱码复查与修复
- 当前判断：项目已经达到“能演示、能部署、能上传 GitHub”的状态

## 当前仍保留的限制
- GitHub Actions 目前做的是“前端构建 + 后端打包”检查，还不是完整集成测试
- Docker 部署链路虽然已具备配置，但仍建议按真实服务器环境再做一轮完整演练
- 前端部分页面仍有继续统一 loading / skeleton / empty 状态的空间
- 后端目前仍以快速迭代为主，部分返回结构后续可继续做 VO 分层

## 下一阶段建议
- 第一优先级：上传 GitHub，固定一版可回滚的 MVP
- 第二优先级：做一次完整部署复盘，确认数据库、Redis、上传目录和 Nginx 在目标服务器上完全稳定
- 第三优先级：继续做前端性能和加载体验统一
- 第四优先级：补最终一轮视觉精修，把项目整理成更成熟的作品集版本

## 结论
- 当前项目不再是课堂级 CRUD 拼装，而是已经具备真实业务主线、权限控制和部署基础的 MVP 版本
- 现在最适合做的事情不是继续无止境补功能，而是固定版本、整理文档、上传 GitHub、完成部署演练，再按优先级继续优化
