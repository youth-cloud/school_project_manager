# 可优化项

## 说明
- 本文件只记录“当前 MVP 可以先不做，但后续继续做会明显提升项目质量”的方向。
- 下面按真实优先级排序，不再按补 CRUD 的思路堆功能。

## 第一优先级

### 1. 完整部署演练
- 用目标服务器环境再完整走一遍前端、后端、MySQL、Redis、上传目录和 Nginx
- 验证 `docker-compose.yml`、`.env.example`、`部署说明.md` 是否真正闭环
- 补充一份更贴近真实上云过程的排错记录

### 2. GitHub CI 升级为更强校验
- 当前已经补了 GitHub Actions，可自动做前端构建和后端打包
- 后续可继续升级为：
- 引入 MySQL / Redis 服务容器
- 跑后端测试或最小集成测试
- 增加 Docker 构建检查
- 增加 PR 必过校验规则

### 3. 前端性能优化
- 继续排查高频页面首屏空白和详情弹窗等待问题
- 优先排查：
- `project_topic`
- `topic_application`
- `project_group_application`
- `project_group`
- `project_group_member`
- `stage_submission`
- `review_record`
- `defense_record`

### 4. Loading / Skeleton / Empty 统一
- 列表首屏 loading 继续统一
- 详情弹窗 skeleton 继续统一
- 空列表、空附件、空详情的 empty 文案继续统一
- “提交中 / 删除中 / 加载中”按钮状态继续统一

## 第二优先级

### 1. 最后一轮界面精修
- 继续优化首页卡片、列表页头部、操作区、详情弹窗层级
- 继续统一按钮间距、图标层次、阴影、留白和色彩细节
- 把当前蓝白风格从“整洁”提升到“更像正式作品”

### 2. 列表页与详情页公共能力继续抽取
- 通用分页状态
- 通用详情头部
- 通用状态标签映射
- 通用时间格式化
- 通用删除确认逻辑

### 3. 选择器体验增强
- 远程搜索
- 大数据量分页选择
- 选项缓存
- 联动筛选
- 更明确的加载态和空态

## 第三优先级

### 1. 业务规则继续产品化
- 建组申请和正式项目组之间增加更明显的跳转联动
- 阶段提交的 `versionNo` 可以考虑改为后端自动生成
- 周报可以继续做更清晰的状态流转
- 答辩安排、答辩记录、成绩记录可以继续增加更严的规则校验

### 2. 认证与安全增强
- 登录失败次数限制
- 多设备登录策略
- 更细的 token 生命周期配置
- 风险登录提醒

### 3. 审计能力升级
- 操作日志可以从手工 CRUD 升级为自动采集
- 自动记录操作人、接口、IP、耗时、结果
- 审计日志建议默认只读

## 第四优先级

### 1. 后端结构化优化
- 逐步把返回结果从实体过渡到更清晰的 VO
- 统一业务错误码和异常分类
- 进一步减少前后端字段耦合

### 2. 数据库约束增强
- 复盘哪些规则适合下沉为联合唯一索引
- 复盘哪些状态适合改为更明确的字典约束
- 评估是否增加归档字段或操作历史字段

### 3. 文件存储升级
- 当前 MVP 使用本地磁盘存储已够用
- 后续如继续生产化，可升级到 MinIO 或云对象存储

## 当前建议
- 如果准备先把项目作为 `MVP` 固定到 GitHub，当前最值得继续投入的是：
- 完整部署演练
- CI 升级
- 前端体验统一
- 最后一轮视觉精修

## 结论
- 这些优化项都值得做，但目前绝大多数都不再阻塞项目演示与发布
- 当前更重要的是先固定版本、整理文档、上传 GitHub，再围绕真实交付体验逐步增强
