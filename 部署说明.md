# 学校实训管理平台部署说明

## 1. 部署目标

本项目采用前后端分离架构：

- 前端：`Vue 3 + Vite + Element Plus`
- 后端：`Spring Boot + MyBatis-Plus + Spring Security + JWT`
- 依赖：`MySQL 8`、`Redis 7`
- 文件上传：后端本地目录或容器卷持久化

当前仓库已补齐以下部署资产：

- 根目录 `docker-compose.yml`
- 根目录 `.env.example`
- 后端 `backend/Dockerfile`
- 前端 `frontend/Dockerfile`
- 前端 `frontend/nginx.conf`
- 后端 `application.yaml / application-dev.yaml / application-prod.yaml`

---

## 2. 推荐部署方式

推荐优先使用 `Docker Compose` 部署，原因：

- 一次性拉起 `MySQL + Redis + 后端 + 前端`
- 环境变量集中管理
- 适合云服务器快速演示
- 后期可平滑迁移到更正式的容器编排方案

---

## 3. 部署前准备

### 3.1 服务器建议

- 操作系统：`Ubuntu 22.04 LTS` 或更新版本
- CPU：2 核起
- 内存：4GB 起
- 磁盘：40GB 起

### 3.2 软件建议

- Docker
- Docker Compose Plugin

### 3.3 端口规划

- `80`：前端站点
- `8080`：后端接口
- `3306`：MySQL
- `6379`：Redis

如果你最终只想暴露前端站点给外网，可以在安全组里仅开放 `80/443`，把 `3306/6379/8080` 保持内网或关闭。

## 4. 数据库初始化

仓库根目录已有两个 SQL 文件：

- `schoolprojectmanager.sql`
- `schoolprojectmanagerwithdata.sql`

建议：

- 纯净环境部署：导入 `schoolprojectmanager.sql`
- 演示环境部署：导入 `schoolprojectmanagerwithdata.sql`

### 4.1 Docker Compose 默认初始化行为

当前 `docker-compose.yml` 已经把根目录的 `schoolprojectmanager.sql` 挂载到 MySQL 的初始化目录：

- 首次启动且 `mysql-data` 还是空卷时，会自动导入库表结构
- 这意味着“纯净环境部署”通常不再需要手工导入 `schoolprojectmanager.sql`

注意：

- 只有在 `mysql-data` 为空时，`/docker-entrypoint-initdb.d` 下的 SQL 才会执行
- 如果你已经启动过一次 MySQL，再改 SQL 文件不会自动重跑
- 如果想重新初始化，需要先删除 MySQL 数据卷再重新启动

### 4.2 Docker MySQL 导入演示数据示例

如果你要部署演示环境并导入 `schoolprojectmanagerwithdata.sql`，建议先启动 `mysql` 服务：

```bash
docker compose up -d mysql
```

然后导入 SQL：

```bash
docker cp schoolprojectmanagerwithdata.sql school-project-mysql:/tmp/init.sql
docker exec -it school-project-mysql mysql -uroot -p${MYSQL_ROOT_PASSWORD} schoolprojectmanager -e "source /tmp/init.sql"
```

如果数据库还不存在，也可以先登录 MySQL 手动创建：

```sql
CREATE DATABASE schoolprojectmanager DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

## 5. 环境变量配置

### 5.1 根目录环境变量

复制一份：

```bash
cp .env.example .env
```

最少需要修改这些值：

- `MYSQL_ROOT_PASSWORD`
- `DB_PASSWORD`
- `JWT_SECRET`
- `APP_CORS_ALLOWED_ORIGINS`

### 5.2 生产环境推荐值

如果你的正式域名是 `https://school-project.example.com`，建议：

```env
APP_CORS_ALLOWED_ORIGINS=https://school-project.example.com
SPRINGDOC_API_DOCS_ENABLED=false
SPRINGDOC_SWAGGER_UI_ENABLED=false
JWT_SECRET=请替换成至少32位以上的随机高强度密钥
```

## 6. 一键启动方式

在项目根目录执行：

```bash
docker compose up -d --build
```

说明：

- 首次启动时，MySQL 会自动初始化 `schoolprojectmanager.sql`
- 如果你希望加载演示数据，请先完成上面的手工导入步骤

查看状态：

```bash
docker compose ps
```

查看日志：

```bash
docker compose logs -f backend
docker compose logs -f frontend
```

## 7. 访问方式

默认访问：

- 前端首页：`http://服务器IP/`
- 后端接口：`http://服务器IP/api/...`
- 上传文件：`http://服务器IP/uploads/...`

当前前端容器内的 `nginx.conf` 已配置：

- `/` -> 前端静态资源
- `/api/` -> 反向代理到后端
- `/uploads/` -> 反向代理到后端上传资源

因此生产环境一般不需要额外给前端设置 `VITE_API_BASE_URL`。

## 8. 非 Docker 部署方式

如果你暂时不想用 Docker，也可以手动部署。

### 8.1 后端

#### 启动前准备

- JDK `25`
- MySQL `8`
- Redis `7`

#### 打包

```bash
cd backend
./mvnw -DskipTests package
```

Windows 下：

```powershell
cd backend
.\mvnw.cmd -DskipTests package
```

#### 运行

```bash
java -jar target/backend-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod
```

你也可以通过环境变量传入：

- `DB_URL`
- `DB_USERNAME`
- `DB_PASSWORD`
- `REDIS_HOST`
- `REDIS_PORT`
- `JWT_SECRET`
- `APP_UPLOAD_BASE_DIR`

### 8.2 前端

#### 安装与打包

```bash
cd frontend
npm ci
npm run build
```

打包产物目录：

- `frontend/dist`

把 `dist` 部署到 Nginx 静态目录即可。

## 9. 上传文件持久化

后端上传目录通过配置项控制：

- 开发环境默认：`./uploads`
- 生产环境默认：`/app/uploads`

Docker Compose 已将其挂载到卷：

- `backend-uploads`

这意味着重启容器后，上传文件不会丢失。

## 10. 生产环境建议

### 10.1 安全建议

- 修改默认数据库密码
- 修改默认 JWT 密钥
- 关闭 Swagger
- 仅开放必要端口
- 给站点补 HTTPS

### 10.2 运维建议

- 为数据库做定期备份
- 为上传目录做定期备份
- 开启日志轮转
- 给 Docker 容器设置自动重启

## 12. 首次部署推荐流程

建议按下面顺序执行：

1. 复制 `.env.example` 为 `.env`
2. 修改数据库密码、JWT 密钥、允许跨域域名
3. `docker compose up -d mysql redis`
4. 等待 `mysql` 和 `redis` 通过健康检查
5. 纯净环境可直接继续下一步；演示环境再手工导入 `schoolprojectmanagerwithdata.sql`
6. `docker compose up -d --build backend frontend`
7. 打开浏览器验证首页、登录、附件访问、上传下载

这样最稳，也最适合你后面上云演示。
