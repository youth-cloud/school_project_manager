## 1. 说明

- 用途：用于本轮“角色校验补强 + Code Review 修复”后的接口回归测试
- 基础地址：`http://localhost:8080`
- 测试方式：建议直接在 Apifox / Postman 按顺序执行
- 测试目标：
  - 确认正常流程仍然可用
  - 确认角色校验真的生效
  - 确认本轮修复点没有回归
- 约定：
  - 看到“记录返回 id”时，请把响应体中的 `data.id` 记下来，供下一步继续使用
  - 预期“失败”时，重点看业务返回是否为失败，以及 `message` 是否符合预期
  - 如果某一步新增成功，会向测试库写入新数据，属于正常现象
